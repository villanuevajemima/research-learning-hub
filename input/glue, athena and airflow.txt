AWS Glue & Athena: Data Management
Course Description
This 4 weeks course focuses on the governance and lifecycle management of a Data Lake. You will learn how to maintain a high-quality Data Catalog, manage schema evolution, enforce data quality rules, use Amazon Athena as a management tool to audit, do orchestration using Airflow and query governed datasets.
Prerequisites
SQL Foundations: Knowledge of DDL (Data Definition Language) like CREATE, ALTER, and DROP.
Data Concepts: Basic understanding of schemas, data types, and primary keys.
AWS Basics: IAM roles and S3 bucket management.

📚 Week 2: Data Transformation & Quality (Glue ETL)
We are diving into AWS Glue, the powerhouse of serverless data integration. We will learn how to write Spark code, enforce data quality, and handle messy data before it breaks our downstream reports.
Day 1: Spark Basics for Glue (DynamicFrames vs DataFrames)
Description: AWS Glue uses Apache Spark under the hood. You need to understand the difference between standard Spark DataFrames and AWS Glue’s custom DynamicFrames. DynamicFrames are designed to handle unexpected schema changes and messy data without crashing your job.
Reference: AWS Glue: The DynamicFrame Class
Day 2: Implementing AWS Glue Data Quality (DQDL)
Description: You shouldn't write hundreds of lines of Python just to check if a column is null. AWS Glue Data Quality uses DQDL (Data Quality Definition Language) to declaratively evaluate your data against business rules in transit.
Reference: AWS Glue: Data Quality Definition Language (DQDL)
Day 3: Handling "Bad Data": Branching logic for Quarantining
Description: When data fails a DQDL check, you don't just delete it. You must write branching logic in your ETL script to route bad records into a "Quarantine" or "Dead Letter" S3 bucket for data stewards to review.
Reference: AWS Big Data Blog: Identifying and Quarantining Bad Data
Day 4: Performance Tuning: Partitioning and Compression
Description: If you dump raw data into a single massive file, Athena will take forever to query it. We will cover logical partitioning (e.g., partitioning by year and month) and compressing data into Parquet formats to optimize performance and reduce AWS costs.
Reference: AWS Prescriptive Guidance: Tuning Apache Spark in Glue
Day 5: Hands-on 2 (Writing a Glue Job with EvaluateDataQuality)

⚙️ Week 3: Orchestration with Amazon MWAA
ETL scripts are useless if someone has to click "Run" every day. We will use Amazon Managed Workflows for Apache Airflow (MWAA) to orchestrate our pipelines.
Day 1: Airflow Concepts: DAGs, Operators, and Sensors
Description: Learn the core building blocks of Airflow. A DAG (Directed Acyclic Graph) is your pipeline blueprint. Operators execute the work (like starting a Glue job), and Sensors wait for conditions (like a file landing in S3).
Reference: Apache Airflow Core Concepts
Day 2: MWAA Environment Setup & Requirements.txt
Description: Setting up MWAA requires specific environment configurations. Learn how to manage Python dependencies using requirements.txt and package custom Airflow plugins for your managed cluster.
Reference: AWS: Working with DAGs and Python dependencies in MWAA
Day 3: Advanced DAGs: Using XComs and Airflow Variables
Description: Airflow tasks run in isolation. If Task A gets a Job ID that Task B needs, you must use XComs (Cross-Communications) to pass that metadata. We will also cover Variables for environment configuration.
Reference: Astronomer Guide: Passing Data Between Tasks (XComs)
Day 4: Error Handling: Retries, Callbacks, and Alerts
Description: Pipelines fail—it's a fact of data engineering. You will learn how to configure task-level retries, set timeouts, and use callbacks to trigger Slack or Email alerts when a critical DAG fails.
Reference: Astronomer Guide: Error Notifications in Airflow
Day 5: Hands-on 3 (Building an MWAA DAG)

📊 Week 4: Analytics, Visualization & Capstone Prep
Data is only valuable if the business can see it. We will unlock advanced Athena queries, establish strict metadata documentation, and visualize pipeline health.
Day 1: Athena Workgroups & Cost Management
Description: Athena charges per terabyte scanned. A bad SQL query can cost a fortune. You will learn how to set up Athena Workgroups to enforce data-scanning limits and isolate query environments between the Data Engineering and Analytics teams.
Reference: AWS: Managing Queries and Controlling Costs with Workgroups
Day 2: SQL for Iceberg: MERGE, UPDATE, and Time Travel
Description: Because we are using Apache Iceberg, Athena becomes more than just a read-only tool. Learn how to execute powerful Data Manipulation Language (DML) queries, like UPSERTs (MERGE INTO) and querying historical snapshots (Time Travel).
Reference: AWS: Querying Iceberg Data Manipulation
Day 3: Data Dictionary & Metadata Best Practices
Description: Documentation is a core engineering requirement. A Data Dictionary acts as the translation layer between technical schemas and business terminology, ensuring users know exactly what cust_act_dt actually means.
Reference: Atlan: Data Dictionary 2026 Implementation Best Practices
Day 4: Building Data Health Dashboards in QuickSight
Description: Operational metadata is critical. You will learn how to connect Amazon QuickSight to your AWS Glue Data Quality outputs to visualize the health of your pipelines (e.g., tracking the percentage of missing values over time).
Reference: AWS Blog: Streamlining Dataset Management & Insights
Day 5: Quicksight Dashboard

Capstone:
Establish a Governed Data Lake for e-commerce transactions. You will manage the ingestion of raw data, enforce strict quality rules, optimize the storage for Athena queries using the Apache Iceberg format, and set up a data management dashboard.
